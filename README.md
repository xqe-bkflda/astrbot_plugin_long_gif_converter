# 长GIF转换插件 for AstrBot

将多张图片等分裁剪后按列组合成多个GIF动画，支持自定义输出宽度、GIF数量和帧停留时间。

## 功能特点

- 支持从消息中提取多张图片（直接发送或引用消息中的图片）
- 将所有图片等比例缩放到统一宽度（可指定）
- 将每张图片垂直等分成 N 个条带
- 按条带位置组合成 N 个 GIF，每个 GIF 包含所有图片的第 K 条带
- 可自定义每个 GIF 的帧停留时间（轮播速度）

## 安装

#方法一: 
1. 将插件文件夹放入 AstrBot 的 `plugins` 目录
2. 重启 AstrBot 
 - 确保 Python 环境已安装 Pillow 和 aiohttp

#方法二: 
 - 在astrbot插件市场下载本插件

## 使用说明

命令格式：
/长gif转换 <gif张数> <裁剪宽度> <轮播速度>

- gif张数：将每张图片垂直切分成多少份，取值范围 2~20
- 裁剪宽度：输出图片的宽度（像素），建议 100~500，图片会等比例缩放至此宽度
- 轮播速度：每个 GIF 中每帧停留的秒数，例如 0.5 表示每秒播放 2 帧

示例：
/长gif转换 5 300 0.3

执行后，插件会从当前消息或引用消息中提取所有图片，生成 5 个 GIF 文件并发送。

## 参数详细说明

| 参数 | 类型 | 说明 |
|------|------|------|
| gif张数 | 整数 | 必须介于 2 到 20 之间，等于每张图片的裁剪份数，也等于生成 GIF 的数量 |
| 裁剪宽度 | 整数 | 输出 GIF 的宽度（像素），所有图片会先等比例缩放到此宽度再裁剪 |
| 轮播速度 | 浮点数 | 每个 GIF 中每帧停留的秒数，总时长 = 轮播速度 × gif张数，不得超过 60 秒 |

## 注意事项

- 所有输入图片的高度必须足够切分成指定的份数，否则生成失败。
- 图片数量不限，但越多处理时间越长。
- 生成的 GIF 文件会暂存在插件数据目录的 temp 文件夹中，发送后 10 秒自动删除。
- 若图片包含透明通道，会自动填充白色背景。
- 建议图片尺寸不要过大（宽度 > 1000 像素可能影响性能）。

## 依赖库

- Pillow：图像处理
- aiohttp：异步下载图片
